# PRD - 活動報名系統 (Event Registration System)

## 1. 專案概述

* **背景與動機**：為了方便活動創辦者能輕鬆發布活動、管理報名資訊，同時避免超額報名的問題，我們需要一個輕量、自動化的專屬活動報名系統。
* **目標用戶**：活動創辦者（主辦方）。
* **核心價值主張**：提供一個直覺且快速的平台，讓主辦方能輕鬆建立包含簡述、時間表等資訊的活動，並在人數額滿時自動阻擋報名，大幅降低人工管理的成本。

## 2. 功能需求

以下為系統的主要功能，並以使用者故事（User Story）格式描述：

1. **可以建立活動**
   * 作為 **活動創辦者**，我希望 **能在系統上填寫活動資訊並發布新活動**，以便 **有一個專屬網頁可以分享給其他人進行報名**。
2. **顯示報名人數**
   * 作為 **活動創辦者**，我希望 **能在活動頁面上看到目前的已報名人數與名額總數**，以便 **隨時掌握這場活動的參與熱度與報名狀況**。
3. **簡述活動內容**
   * 作為 **活動創辦者**，我希望 **能填寫和展示活動的詳細簡述與說明文字**，以便 **讓潛在報名者能清楚了解活動的目標與內容細節**。
4. **可顯示活動時間表**
   * 作為 **活動創辦者**，我希望 **能設定並在頁面上顯示活動的時間表（如活動日期、開始與結束時間）**，以便 **報名者確認自己是否有空參加該活動**。
5. **人數滿即關閉報名**
   * 作為 **活動創辦者**，我希望 **系統在報名人數達到設定的上限時，自動關閉報名功能並顯示已額滿**，以便 **避免超額報名造成的場地或資源分配問題**。

## 3. 非功能需求

* **技術限制**：
  * 後端框架採用 `Flask`。
  * 前端頁面渲染使用 `Jinja2` 模板引擎。
  * 資料庫使用 `SQLite` 進行輕量級資料儲存。
* **效能與安全考量**：
  * **併發控制 (Concurrency)**：確保在極短時間內多人同時報名時，資料庫的更新能正確扣除剩餘名額，避免超爆人數。
  * **資安防護**：使用者輸入的所有欄位（如活動名稱、簡述等）均需過濾防範 XSS (跨站腳本攻擊) 與 SQL Injection 等常見網站安全風險。

## 4. MVP 範圍 (Minimum Viable Product)

* **Must Have (必須有)**：
  * 活動創立頁面（可設定名稱、簡述、時間表、人數上限）。
  * 活動詳情與報名頁面（顯示活動內容、目前已報名人數/名額）。
  * 基本報名功能（填寫簡單資訊送出）。
  * 人數額滿即關閉表單，防止進一步報名。
* **Should Have (應該有)**：
  * 報名成功頁面與提示。
  * 讓主辦方可瀏覽「所有報名者清單」的介面。
* **Nice to Have (加分項)**：
  * 報名者可「取消報名」，將名額釋放給其他人。
  * 使用者登入/註冊機制管理不同主辦方建立的活動。
  * 活動海報/圖片上傳功能。
